# Vault Structure

Everything devnexus creates in the Obsidian vault, what it's for, and when it's updated.

## Vault Layout

```
{projectName}-vault/
├── .obsidian/                    # Obsidian config (auto-generated)
│   ├── app.json                  # Settings: update links, attachment folder
│   ├── core-plugins.json         # Standard plugins
│   ├── community-plugins.json    # ['obsidian-git']
│   └── plugins/obsidian-git/     # Auto-sync config (1 min commit/pull)
├── .gitignore                    # Excludes workspace.json, cache
├── MOC.md                        # Map of Content — entry point
├── ARCHITECTURE_OVERVIEW.md      # System design
├── API_CONTRACTS.md              # Endpoint definitions
├── DECISIONS.md                  # Rejected approaches, non-obvious choices
├── SESSION_LOG.md                # Session handoff notes
├── GRAPH_REPORT.md               # Structural analysis
├── NODE_INDEX.md                 # Full symbol table (generated by devnexus index)
└── nodes/                        # Per-community symbol files (generated by devnexus index)
    └── {community}/
        ├── _COMMUNITY.md         # Hub nodes, members, internal call graph
        └── {symbol}.md           # Callers, callees, cross-references
```

## Core Files

### MOC.md — Map of Content

The entry point. Agents read this first to orient themselves.

Contains:
- Project name and description
- Table of tracked repos with paths and tech stacks
- Links to every other vault file

Updated by `devnexus init` and `devnexus add`.

### ARCHITECTURE_OVERVIEW.md — System Design

High-level view of the project:
- Project description and tech stack
- How repos connect to each other
- Key data structures
- Key communities and god nodes (injected by `devnexus index` between `<!-- devnexus:graph:start -->` and `<!-- devnexus:graph:end -->` markers)
- Known gaps

Updated manually during sessions. The code graph section is auto-generated by `devnexus index`.

### API_CONTRACTS.md — Endpoint Definitions

The **final authority** on API shapes. Every endpoint definition includes:
- Route and HTTP method
- Request body (fields, types, required/optional)
- Response shape (fields, types)
- Error responses (status codes and meanings)

Enforced by the pre-push hook — if API code changes but this file doesn't, the push is blocked.

### DECISIONS.md — Decision Log

Reverse-chronological log of rejected approaches and non-obvious choices.

Format:
```markdown
## YYYY-MM-DD — Title (by name)

Two sentences. What was tried/decided and why.
```

Updated by agents during sessions when approaches fail or non-obvious choices are made.

### SESSION_LOG.md — Session Handoff

Two-line handoff notes so the next session picks up where the last one left off.

Format:
```markdown
## YYYY-MM-DD — Summary (by name)

Line 1: What was done.
Line 2: Where things left off.
```

Updated at the end of every session.

### GRAPH_REPORT.md — Structural Analysis

Generated by `devnexus index`. Contains:
- God nodes (high betweenness centrality symbols)
- Bridges (sole edges connecting communities)
- Knowledge gaps (thin communities, low cohesion, oversized clusters)
- Graph diff from previous index (new/removed god nodes, community changes)

### NODE_INDEX.md — Symbol Table

Generated by `devnexus index`. Contains:
- God nodes table (symbol, betweenness centrality, edge count, cross-community reach)
- Communities table (name, hub nodes, member count, cohesion)
- Full symbol table (every indexed symbol with tier, BC, edges, file, repo)

### nodes/ — Per-Community Files

Generated by `devnexus index`. One directory per detected community:
- `_COMMUNITY.md` — Hub nodes, all members, internal call graph
- `{symbol}.md` — Individual symbol files with callers, callees, and cross-references

## Generated vs Manual Files

| File | Generated By | Manually Updated |
|------|-------------|-----------------|
| MOC.md | `devnexus init`, `devnexus add` | Rarely |
| ARCHITECTURE_OVERVIEW.md | `devnexus index` (partial) | Yes — agents update during sessions |
| API_CONTRACTS.md | `devnexus init` (template) | Yes — agents update when APIs change |
| DECISIONS.md | `devnexus init` (template) | Yes — agents update during sessions |
| SESSION_LOG.md | `devnexus init` (template) | Yes — agents update at session end |
| GRAPH_REPORT.md | `devnexus index` | No |
| NODE_INDEX.md | `devnexus index` | No |
| nodes/ | `devnexus index` | No |
